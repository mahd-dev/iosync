iosync

Nodejs module that synchronizes data in real-time between server and different connected clients, it includes “data binding”, “scopes”, “session and login”, “query”, “redirect”, “middlewares”, etc...

Principle :

It's based on client-server architecture so client binds to specific variables names and when that variable was changed then the server the âpatch_processorâ function to store it and diffuse that change to other binded clients based on it's scope and connected user




Here is an example of scope principle and how variables works virtually, it supposes that there is a json object contains all data on server and every client has a real-time synchronized copy of the server json object.

API :

Server side :

bind ( { path, scope, provider, patch_processor } )

Description :
binds to a specific âpathâ and define it's âscopeâ and controllers

Attributes :

Option

type

required

description

path

string

yes

The path to bind

Example:
â/articles/541â

scope

string or array or object

yes

Defines who can see the data for this path and who can edit them

It can be in 4 different cases:

1: âpublicâ : anyone can view and edit even not connected users

2: âuserâ : a user specific, so each user has it's own data shared between clients connected as this user

3: array or object : only users having one of listed capabilities can access

3.1 : [ ] : an array of capabilities names this will apply for both read and write access

3.2 : { read : [ ], write : [ ] } : dividing who can read from who can write

4 : âsessionâ : client specific scope

When a client needs to read this path's data, this function will be called provided with the client's session and params expecting to callback with data
